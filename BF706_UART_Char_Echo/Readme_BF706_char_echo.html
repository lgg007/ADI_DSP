1. Description

Demonstrates basic UART operation using charecter echo.

2. Processor

  • ADSP-BF707

3. Overview

This example uses the BF707 UART driver to do character echo onto PuTTY/TeraTerm. This example uses the UART driver blocking mode buffer transaction APIs to read and write data to the UART peripheral.

The device drivers can operate in one of three modes.

  1. Non-blocking
  2. Blocking
  3. Callback

This example demonstrates the UART driver being operated in both Blocking mode. For more information on driver modes of operation please refer to the driver help pages.

By default the example configures the UART to operate in interrupt mode. The user configurable macro USE_DMA_MODE can be uncommented to configure UART to use DMA for data transmit and recieve.

The user configurable macros can be found in char_echo.h file. Please read the following section for available user configurable macros.

4. Hardware Setup

Connect Micro USB cable to J2 to P6 on ADSP-BF707 board one end and other end to PC. Open PuTTY/TeraTerm and configure it with the settings in Table 1.

The appropriate COM port number can be found in the Windows Device Manager under the Ports (COM & LPT) node.

Table 1: COM Settings
ConfigurationValue
Bits per second9600
Data bits8
ParityNone
Stop bits1
Flow ControlNone

6. Software switch settings

The following default UART-related switches have been set:

  • ~UART0CTS_EN set to 0
  • ~UART0RTS_EN set to 0
  • ~UART0_EN set to 0

See SoftConfig_BF707.c for more information.

Please refer to the ADSP-BF707 EZ-KIT Lite Evaluation System Manual for information on software-controlled switches.
